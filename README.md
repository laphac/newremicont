# newremicont
Проект ставил задачу заменить по функционалу автоматический регулятор общего воздуха к котлу (РОВ) котлоагрегата ТГМЕ-464, вместо устаревшего микроконтроллера Ремиконт Р-130. Соединение с устоявшейся схемой контакт в контакт, без переделок. От Р-130 брались только разьемы на торцевой панели.

Описание логики работы техпрограммы в файле README.pdf
Схема и подключение блоков КБС-2 КБС-3 в image
Модель кейса (коробка) для плат в 3d-box
Скриншоты рабочего окна для настроки коэфициентов - screen

Список оборудования:

Описание процесса:
Контроллер ESP32-C3 работает под MicroPython. Краткая инструкция по прошивке:
Шаг 1: Установка ПО на ПК
1. Установите Python: Скачайте и установите последнюю версию Python с python.org.
2. Установите esptool: Откройте командную строку (CMD/Terminal) и введите pip install esptool.
3. Проверьте esptool: Введите esptool.py -h или esptool.exe -h (для Windows) для проверки.
4. Установите драйверы: Для некоторых плат ESP32 (например, с чипом CH9102X) могут потребоваться специфические драйверы USB-UART.
Шаг 2: Загрузка прошивки MicroPython
1. Перейдите на сайт MicroPython и скачайте последнюю прошивку .bin для ESP32-C3. Версий много не ошибись.
Шаг 3: Прошивка ESP32
1. Подключите плату: Соедините ESP32 с компьютером USB-кабелем.
2. Определите порт: Узнайте COM-порт вашей платы (в Диспетчере устройств Windows или через esptool.py port или в настройках IDE).
3. Перед записью новой прошивки лучше отформатировать ESP32 командой в терминале. "esptool --port <COM-порт> erase_flash"
4. Запишите прошивку: В терминале введите команду (заменив путь к файлу и порт):
"esptool --port <COM-порт> --baud 460800 write_flash --flash_size=detect 0 <путь_к_файлу_прошивки>.bin"
Шаг 4: Работа с MicroPython
1. Используйте IDE: Thonny или Mu Editor (не поддерживает некоторые китайские платы)
2. Программа которая вызывается при старте имеет имся boot или main.
